Sandbox 少了 73 个文字元素（41% 丢失）。主要缺失页：
- 第 4 页：Golden 36 文字 vs Sandbox 部分匹配（对比脚本按页码对齐，错位后显示 35 个缺失）
- 第 5/7/8 页：章节页大量文字缺失（"工程化能力"、"演示模式"、"v2.12 新增"等）

**可能原因**：这些文字在 Sandbox 的 flat_extract 中被嵌套在 container 的 grandchildren 里，但渲染时可能被跳过或位置异常（如 y 为负值）导致未渲染。需要独立验证。

## 根因分析

### Bug 1：Grid 子容器内 background shape 撑高容器（影响 Y 偏移）

**现象**：Sandbox 的 Grid 子容器高度 2.32"，Golden 的实际 grid 区域高度 1.69"。多出 0.63"。

**原因**：`build_grid_children` 中，每个 grid cell 的 nested container 第一个 child 是 background shape（h=1.86"），它参与了 flex-column 高度计算，撑大了整个容器。

**Golden 行为**：Golden 也有 y=3.23" h=2.38" 的 background shape，但它通过 `position: absolute` 或独立的 layer div 实现层叠，不参与文字流。**Sandbox 的 shape 是通过 `flat_extract` 递归提取的，HTML DOM 中 shape 的 div 是 grid cell 的直接子元素，导致被当作普通子节点处理。**

**已知不确定性**（Codex review 指出）：需要确认 Golden 的 shape 隔离机制具体是什么（CSS position？z-index？还是 Playwright 提取代码的主动过滤？）。目前只有假设，没有代码级验证。

**影响**：Grid 容器的 `height` 被错误计算为 2.47"（实际应 ~1.69"），导致容器后面的元素被推远。同时 `justifyContent: center` 的垂直居中计算基于错误的内容高度，进一步偏移。

**修复方案**：在 `build_grid_children` 中，对 nested container 内的纯 background shape（无文字、有渐变/背景色、高度 > 1.0"）标记为不参与高度累加。

### Bug 2：Grid 内容起始 Y 缺少内部间距

**现象**：Grid 内第一个文字条目从 y=0.00"（相对 container）开始，Golden 实际从 container 内部 y≈0.47" 开始（Golden container 从 y=3.23" 开始，第一个文字 y=3.44"，差 0.21"；但 Golden 的 container 顶部有 shape 在 y=3.07-3.23"）。

**原因（待验证）**：HTML DOM 中 grid 容器（`.cols2`）上方有一个 `.divider` 元素（分割线），Golden 中它占据了 y=3.07"-3.23" 的空间（h=0.03" + 间隔 0.16"）。Sandbox 的 `flat_extract` 可能没有正确提取分割线，或者提取后没有参与 Y 堆叠。

**修复方案（待验证 DOM 结构后确定）**：确认 divider 是否被提取，如果被提取是否正确参与了 Y 累加。

### Bug 3：标题起始 Y 偏移 0.48"（非网格页）

**现象**：非网格页（第 2、3 页）标题比 Golden 低 0.47-0.48"。网格页（第 4 页）标题偏移更大（-1.04"），是 Bug 1 的叠加效应。

**数据**：
- Golden 标题 y ≈ 2.15"
- Sandbox 标题 y ≈ 1.67"
- 差值 0.48"

**来源分析**：Golden 的 y=2.15" = slide `padding-top: clamp(28px, 4vw, 56px)` = 56px = 0.52" + 内容区内部偏移 1.63"。Sandbox 的 `internal_margin` = 0.52"（从 paddingTop 派生），标题实际从 0.52" + divider(0.03") + 1.12" = 1.67" 开始。**中间差了 0.48"，这个值的来源尚未确定。** 可能是 Playwright 版本对 content-root div 的额外 padding，或 DOM 提取时的特殊处理。

**状态**：待调查，标记为 P2。Bug 1 修复后再重新测量。

### Bug 4：第 9 页（精修模式）溢出

**现象**：Sandbox 第 9 页拆成 2 页（精修模式 + 两种规划深度），导致后续页面对齐错位。

**原因（推测，待验证）**：可能与 Bug 1 相同 — 文字高度偏大 + shape 撑高容器。但精修模式不是 grid 布局，可能是纯 flex-column，需要单独确认溢出触发点。

**状态**：P1，独立验证，不假设自动解决。
